# GenON LIFE AICC 데모 시연 스크립트 v2

단일 시연 흐름. 각 화면의 패널·카드 구성(어떤 기능이 있는지)을 먼저 설명하고, 그 위에서 무엇을 조작·주목하는지 정리합니다. (노션 붙여넣기용 — 표·코드블록·구분선 없이 제목·불릿로만 작성)

**전체 흐름**: AI 상담 홈 → (김민준) 콜 받기 → 실시간 상담 → SMS 안내 → 상담 이력 조회 → 접촉이력 등록 → 이력 반영 확인 → 검수 완료 필터 → (정해린) AI 검수 결과 → 검수 결과 화면

**등장 인물**
- 상담사 김제나 (gena.kim) — 실시간 상담·후속처리
- 관리자 박관리 (park.admin) — 모니터링·코칭·검수
- 고객 김민준 — 실시간 상담(추가서류 → 자동이체 → 해지환급금)
- 고객 정해린 — 검수 대상(환급률 오안내 + 보장 종료 고지 누락)

**데모 리셋**: 페이지 새로고침 시 김민준 후속업무(접촉이력·SMS)가 초기 상태로 돌아갑니다. 처음부터 다시 시연할 때 새로고침하세요.

## STEP 1 — AI 상담 홈

상담사 업무의 출발점. 좌측은 개인 대시보드 + 업무 어시스턴트, 우측은 실시간 콜/이력 사이드바.

**화면 구성**
- 상담 KPI 대시보드(상단): 오늘의 상담 목표율(14/60 진행바), 처리 건수·고객만족도, 후처리 대기 N건(앰버) — 오늘 내 업무 현황.
- 나만의 업무 어시스턴트: 그라데이션 테두리 검색창(약관·규정·스크립트 검색) + 예시 질문 리스트.
- 바로가기: 자주 쓰는 업무 화면 4종 카드(아이콘 그리드).
- 콜 인입 패널(우측 상단): 수신 대기 ↔ 인입 중. 인입 시 대기 콜·대기 시간·채널 수치, 통화 받기 버튼, 고객 정보 카드(고객명/번호/등급/채널 + AI 예상 문의).
- 오늘의 고객 상담 이력(우측 하단): 최근 상담 목록(고객·요약·상담번호·시간 + 후처리 필요 태그). 클릭 시 해당 후속업무 화면으로 이동.

**조작 / 주목**
- 진입 약 3초 후 콜 인입(점 깜빡임) → 약 1초 뒤 고객(김민준) 정보 자동 로딩(AI 예상 문의 포함).
- 통화 받기 클릭 → 실시간 상담 화면.
- 멘트: "콜을 받기도 전에 누가, 어떤 이력의 고객인지, 무엇을 물어볼지까지 AI가 미리 띄워줍니다."

## STEP 2 — 실시간 고객 상담

상단 컨텍스트 바 + 4분할 작업 공간. STT·가이드·지식검색(RAG)·업무정보가 발화에 맞춰 동시에 반응하는 AICC 핵심 화면. 진입하면 STT 대화가 한 줄씩 떠오르며 화면 곳곳이 순서대로 반응합니다(자동재생, 별도 조작 불필요).

**화면 구성**
- 컨텍스트 바(상단): 고객·상담ID + 이력 플래그(실효 이력·갱신 안내장·보험금 접수중).
- ① STT 패널: 화자별 말풍선 실시간 표시. 고객 발화의 키워드 노란 하이라이트, 말풍선 다중선택 → 복합 질의.
- ② 상담 가이드 패널: FCR 필수 안내 확인(본인확인·녹취고지 등 자동 체크), 추천 상담 스크립트(대화 유형별), 대화 키워드·FAQ(추출 키워드 + 추천 질의).
- ③ 상담지식 어시스턴트(RAG) 패널: 약관·업무 기준 지식 질의 전용. 질문 → 탐색 4단계 → 답변(요약·근거·출처·업무기준) + FAQ 칩 + 입력창.
- ④ 우측 패널(상하 분할):
  - 업무정보: 계약 선택 + 처리계 연동 데이터(청구 서류 제출 현황·처리 가능 여부·예상 환급금/한도·유의사항).
  - Next Step·작업 추천: 토픽별 다음 업무 화면 + 후속 작업(SMS 안내·접촉이력 등록).
- 관리자 코칭 패널(우측 하단): 관리자 실시간 청취·코칭 메신저.

**핵심 — 역할 분리**: RAG = 기준/약관(일반 기준), 업무정보 = 고객의 실제 현황(제출/미제출·처리 가능 여부 등 연동 데이터). 같은 추가서류 질문에서도 RAG는 약관상 필요 서류 목록을, 업무정보는 고객의 실제 제출 현황(진료비계산서 제출 / 세부내역서 미제출)을 보여줍니다.

**발화 흐름 상세 (자동재생 6스텝)**

STEP 0 — 도입·본인확인 (약 4.0초)
- 상담사: "안녕하세요, 제논라이프 상담사 김제나입니다. 통화는 녹취되며, 본인 확인과 개인정보 수집·이용 동의가 필요합니다."
- 컨텍스트 바에 이력 플래그 3종 등장: 실효 이력 3건 / 갱신 안내장 수령 / 보험금 접수 중.
- FCR 필수 안내 자동 체크: 본인 확인 → 녹취 고지 → 개인정보 동의 (에메랄드 펄스).
- 주목: 콜 인입 즉시 고객 정보·이력 플래그·필수 안내가 자동 세팅.

STEP 1 — 고객 1차 질의: 보험금 청구 추가서류 (약 5.6초)
- 고객: "보험금 청구를 했는데 추가 서류가 필요하다고 문자가 왔어요…" (버블 내 '추가 서류' 노란 하이라이트)
- 키워드 '추가서류' 추출 → 파란 펄스 자동 선택.
- RAG 질의 "실손 보험금 청구 시 필요 서류 기준" → 약관상 서류 기준 답변(진료비계산서·세부내역서 기본, 고액·특정 질환 시 진단서 / 출처: 실손 약관 제12조).
- 업무정보(계약 SL-2048-5521 실손 선택) → 청구 서류 제출 현황: 진료비계산서 제출 완료, 세부내역서 미제출.
- 주목: RAG는 기준을, 업무정보는 제출 현황을 분담 — 역할 분리 시연.

STEP 2 — 상담사 안내: 세부내역서 (약 4.0초)
- 상담사: "확인해 보니 진료비 세부내역서가 미제출 상태입니다. 병원 원무과 발급 후 앱으로 제출해 주시면 됩니다."
- Next Step 강조: 보험금 청구 진행조회 / 추가서류 접수 등록 + 후속 작업(SMS·접촉이력) 추천 점등.

STEP 3 — 고객 2차(복합) 질의: 자동이체 계좌 변경 (약 5.6초)
- 고객: "그리고 보험료 자동이체 계좌도 바꾸고 싶은데 가능할까요?"
- 키워드 '자동이체 계좌변경' 추출 → 자동 선택.
- RAG 질의 "보험료 자동이체 계좌 변경 기준" → 변경 기준(본인 명의 원칙, 출금 5영업일 전 신청 당월 적용, 카드납 전환 가능 / 출처: 수금·자동이체 업무지침 제8조).
- 업무정보(계약 SL-1180-3372 종신 선택) → 처리 가능 여부, 월보험료/예상 환급금/변액, 유의사항.
- 주목: 복합 질의를 1차 청구 건과 분리 처리, 계약 기반 처리 가능 여부 판단.

STEP 4 — 상담사 처리 가능 확인 (약 4.0초)
- 상담사: "네, 본인 명의 계좌로 변경 가능합니다. 처리 가능 여부와 유의사항 확인해 드리겠습니다."
- 업무정보 유의사항(정상 납입 중·실효 후 부활 이력·해지방어) 사전 조회.

STEP 5 — 고객 3차 질의: 해지환급금 (약 5.6초)
- 고객: "예전에 가입한 종신보험을 해지하면 환급금이 얼마나 나오는지도 궁금해요."
- 키워드 '해지환급금' 추출 → 자동 선택.
- RAG 질의 "종신보험 해지환급금 안내 기준" → 안내 기준(산출식, 보장 종료·재가입 제한 고지, 해지방어 우선 / 출처: 종신 약관 제30조).
- 업무정보(종신) → 해지환급금 예상액·변액 정보, 해지 가능(해지방어 안내 필수).
- 핵심 장면: 민감한 해지 안내 시점에 관리자 코칭 패널이 자동으로 펼쳐지고 '실시간 청취 중' + 코칭 메시지 도착("보장 종료·재가입 제한을 먼저 고지하세요" 등).
- Next Step 강조: 해지환급금 조회 / 해지방어 캠페인.

종료 — 통화 종료 → 헤더 '내 상태'가 통화 대기로 자동 전환 → 우측 후속작업의 SMS 안내 발송 카드 점등.

**스텝 내부 타이밍 메모**: 말풍선 노출 → 약 0.6초 후 키워드 자동 선택 → RAG 질문 전송(약 1.3초) → 답변 출력(약 2.5초). STT가 길어지면 자동 스크롤업.

**조작**: 점등된 SMS 안내 발송 카드 클릭 → SMS 화면.

## STEP 3 — SMS 안내 작성

상담 내용을 근거로 안내 문자를 만들어 발송하는 3분할 화면.

**화면 구성**
- 좌: 근거 패널 — 어시스턴트가 정리한 안내 근거 + 추천 질의 칩(해지환급금 산출 근거 등).
- 가운데: 초안 생성 브리지 — 좌측 근거 → 초안 생성 변환 버튼(화살표).
- 우: 문자 안내 초안 패널 — 어시스턴트 근거 기반·상담사 편집 가능한 초안 + 발송하기.

**조작 / 주목**
- 초안 생성: 생성 전 화살표가 깜빡이며 유도(1회 생성 후 멈춤) → 근거 기반 초안 자동 작성.
- 발송하기 → "발송 중…" → 발송완료(파랑 유지)·편집창 비활성 + "김민준 고객님 000-000-0000로 발송되었습니다".

## STEP 4 — 상담 이력 조회

상담 테이블 탐색 + 행 선택 시 우측 상세 카드.

**화면 구성**
- 필터 바: 기간(최근 7일 등), 검수 필터(전체/검수 대기/검수 완료), 통화별·고객별 뷰, 검색창.
- 상담 테이블: 상담일시·고객·채널·요약 + 접촉이력 / SMS / AI 검수 / 휴먼 검수 상태 컬럼.
- 상세 카드(우측 드로어, 행 클릭 시): 가입 상품 · 상담 요약(대화 키워드·원문 보기·요약·관리자 코칭 이력·다음 예상 문의) · 후속 작업(SMS·접촉이력) · 고객 상태·필요 조치 · 동일 고객 상담 이력 · 상담 내용 검수(결과 보기).

**조작 / 주목**
- 김민준 행: SMS = 발송완료(방금 반영), 접촉이력 = 미등록.
- 미처리 셀(미등록/발송요청/감지)에 호버하면 화살표(→) → 클릭 시 해당 처리 화면으로 바로 이동.
- 조작: 김민준 '미등록' 셀 클릭 → 접촉이력 등록.

## STEP 5 — 접촉이력 등록

종료 상담을 분석해 유형 분류·요약 초안을 만들고 이력으로 등록하는 3분할 화면.

**화면 구성**
- 상담 개요(좌): 전체 요약 + 유형별 요약 + 원문 통화 내용 보기.
- 접촉 유형 분류(중): 의도 분석 기반 유형 탭 + 유형별 요약 초안 편집.
- 접촉 이력(우): Tele-Pro 형식 접촉이력 입력/등록.

**조작 / 주목**
- 진입 시 분석 연출(요약·유형 초안 단계적 생성).
- 접촉이력 등록 → 등록 완료 전환 + 등록 내용이 상담 개요에 반영(반영 로딩 후 갱신).

## STEP 6 — 상담 이력 반영 확인

- 조작: 다시 상담 이력 조회 진입.
- 주목: 김민준 행 접촉이력이 등록완료로 갱신(SMS 발송완료와 함께 후속업무 처리 완료).
- 멘트: "처리 결과가 이력에 즉시 반영됩니다."

## STEP 7 — 검수 완료 필터 → (정해린) AI 검수 결과 보러가기

- 조작: 필터 바에서 '검수 완료' 선택 → AI 일괄 검수가 끝난 과거 건만 표시.
- 주목: 정해린(CL-20260513-027) 행 — AI 검수 = '오안내 1·누락 1' 태그 + 심각 배지. 휴먼 검수는 상담사 화면 기준 '조치 필요'로 표기.
- 진입: AI 검수 셀에 호버 → 화살표 → 클릭(또는 행 클릭 후 상세 카드 결과 보기 →) → 검수 결과 화면.

## STEP 8 — 상담 검수 결과

AI 1차 검수와 휴먼(관리자) 2차 검수를 한 화면에 담은 검수 리포트.

**화면 구성**
- 고객 정보 헤더(상단): 정해린 · 고객번호 · 상담일시 + 검토자 박관리 + 관리자 2차 검토 배지.
- 좌: 상담 대화 패널 — 검수 대상 통화 원문(화자별).
- 우: 검수 결과 패널
  - AI 검수 종합: 감지 요약(환급률 오안내 + 보장 종료 고지 누락 / 심각).
  - 탭: 오안내 검수 / 안내 누락 검수 — 탭별 판정 카드(평가 사유) + 판정 근거(약관·내규) + 매뉴얼 수행 여부.
  - 휴먼 검수: 역할에 따라 분기.

**역할별 휴먼 검수 (중요)**
- 관리자 화면: 검수 판정 선택(조치 필요 / 관리자 이관 / 조치 완료 / 검토 완료) + 관리자 코멘트 직접 작성 + (조치 필요 시) 후속 조치 선택(고객 재안내 전화 / 정정 문자 재발송) + 검수 결과 저장.
- 상담사 화면: 관리자 검수 결과를 읽기 전용으로 열람(관리자 코멘트 + 조치 상태). 아직 미검토 건은 '관리자 검토 대기'로 표시.
