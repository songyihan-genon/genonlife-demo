# GenON LIFE AICC 포털 — 데모 시연 가이드

콜센터 상담사/관리자 업무 흐름을 보여주는 데모입니다. 두 가지 시나리오를 순서대로 시연합니다.

- **시나리오 1 — 김민준**: 실시간 상담 → SMS 안내 → 접촉이력 등록 (상담사 관점)
- **시나리오 2 — 정해린**: 상담 검수 결과 확인 (관리자 관점)

---

## 0. 사전 준비

| 항목 | 내용 |
| --- | --- |
| 실행 | 프론트엔드 dev 서버 기동 후 `http://localhost:3000` 접속 |
| 시작 화면 | `/login` — 계정 유형 선택 후 로그인 |
| 상담사 계정 | `gena.kim` (김제나) — 실시간 상담·후속처리 시연용 |
| 관리자 계정 | `park.admin` (박관리) — 상담 검수 시연용 |

> **데모 리셋**: 페이지를 **새로고침**하면 김민준 건의 후속업무(접촉이력/SMS)가 초기 상태(미등록·미발송)로 리셋됩니다. 시나리오를 처음부터 다시 시연하려면 새로고침하세요. (시연 중 화면 이동은 상태가 유지됩니다.)

## 시나리오 1 — 김민준: 실시간 상담 → 후속처리

> 상담사 계정(김제나)으로 로그인한 상태에서 진행합니다.

### 흐름 요약
```
홈(콜 인입) → 통화 받기 → 실시간 고객 상담 → SMS 안내 작성 → 발송완료
   → 상담 이력 조회(접촉이력 미등록) → 접촉이력 등록 → 등록완료
```

### 단계별 조작

| # | 조작 | 화면 | 화면에서 벌어지는 일 |
| --- | --- | --- | --- |
| 1 | 홈 진입 후 대기 | **AI 상담 홈** | 콜 인입 패널이 약 3초 후 "수신 대기 → 인입 중"으로 전환되고, 약 1초 뒤 고객정보(김민준)가 로딩됩니다. |
| 2 | **[통화 받기]** 클릭 | → **실시간 고객 상담** | STT 대화가 실시간 스트리밍되며 길어지면 자동 스크롤됩니다. 우측에 추천 답변(RAG), 대화 키워드, 고객 정보(실효 이력·키워드 태그), 관리자 코칭 패널이 표시됩니다. |
| 3 | 대화 진행(자동 연출) | **실시간 고객 상담** | 시나리오가 끝나면 통화가 종료되고 헤더의 '내 상태'가 자동으로 **통화 대기**로 바뀝니다. 우측 후속작업의 **'SMS 안내 발송'** 카드가 점등됩니다. |
| 4 | **'SMS 안내 발송'** 카드 클릭 | → **SMS 안내 작성** | 상담 요약 기반 문자 초안 작성 화면으로 이동합니다. |
| 5 | 톤 칩 선택 → **[초안 생성]** | **SMS 안내 작성** | 좌측 근거를 바탕으로 문자 초안이 생성됩니다. (생성 전 화살표가 깜빡이며 유도, 1회 생성 후 멈춤) |
| 6 | 초안 확인/편집 → **[발송하기]** | **SMS 안내 작성** | "발송 중…" 후 버튼이 **[발송완료]**(파랑 유지)로 바뀌고 편집창이 비활성화됩니다. *"김민준 고객님 000-000-0000로 발송되었습니다"* 안내가 표시됩니다. |
| 7 | 사이드바 **상담 이력 조회** 진입 | → **상담 이력 조회** | 기본 '최근 7일' 테이블. 김민준 행: **SMS = 발송완료**(방금 발송), **접촉이력 = 미등록**(아직 미처리). |
| 8 | 테이블 **'미등록'** 셀 호버 → 클릭 | → **접촉이력 등록** | 셀에 호버하면 화살표(→)가 나타나며, 클릭하면 상세 카드를 거치지 않고 바로 등록 화면으로 이동합니다. |
| 9 | 등록 화면 확인 → **[접촉이력 등록]** | **접촉이력 등록** | 3분할 화면(좌: 상담 개요 / 중: 접촉 유형 분류 / 우: 접촉 이력). 종료 상담을 분석해 유형 분류·요약 초안을 자동 생성하고, 등록 시 **등록 완료**로 전환되며 상담 개요에 반영됩니다. |
| 10 | 다시 **상담 이력 조회** | **상담 이력 조회** | 김민준 행의 접촉이력이 **등록완료**로 갱신되어 있습니다. |

### 보조 동선
- **상세 카드(드로어)**: 테이블 행을 클릭하면 우측에 상담 요약·후속 작업·상담 내용 검수가 정리된 카드가 열립니다. 후속 작업 카드의 버튼으로도 SMS/접촉이력 화면에 진입할 수 있습니다.

## 시나리오 2 — 정해린: 상담 검수 결과 확인

> 관리자 계정(박관리)으로 로그인하면 검수자 관점이 자연스럽습니다. (상담사 계정에서도 동일 화면 확인 가능)

### 흐름 요약
```
상담 이력 조회 → '검수 완료' 필터 → 정해린 건(오안내·누락 감지)
   → AI 검수 셀 호버/클릭 → 상담 검수 결과
```

### 단계별 조작

| # | 조작 | 화면 | 화면에서 벌어지는 일 |
| --- | --- | --- | --- |
| 1 | **상담 이력 조회** 진입 | **상담 이력 조회** | 테이블 상단 검수 필터에서 **'검수 완료'**를 선택합니다. (AI 일괄 검수가 끝난 과거 건만 표시) |
| 2 | 정해린 건 확인 | **상담 이력 조회** | 정해린(CL-20260513-027) 행: AI 검수 = **'오안내 1·누락 1'** 태그 + **심각** 배지, 휴먼 검수 = **조치 필요**. |
| 3 | **AI 검수 셀** 호버 → 클릭 | → **상담 검수 결과** | 셀에 호버하면 화살표(→)가 나타나며, 클릭 시 해당 건의 검수 결과 화면으로 바로 이동합니다. |
| 4 | 검수 결과 확인 | **상담 검수 결과** | 상단에 고객 정보 헤더 + **관리자 2차 검토 완료** 배지, 'AI 검수 종합' 요약이 표시됩니다. |
| 5 | 탭 전환 | **상담 검수 결과** | **오안내 검수 / 안내 누락 검수** 탭으로 구분. 각 탭에 판정 카드(평가 사유)·판정 근거·매뉴얼 수행 여부가 정리됩니다. |
| 6 | 하단 휴먼 검수 확인 | **상담 검수 결과** | 관리자 코멘트와 조치 상태(조치 필요/완료/이관)가 검토자(박관리)와 함께 표시됩니다. |

### 보조 동선
- **상세 카드 경유**: 테이블 행 클릭 → 우측 상세 카드의 **'상담 내용 검수'** 박스에서 **[결과 보기 →]**를 눌러도 동일한 검수 결과 화면으로 이동합니다.

## 부록 — 화면별 한눈 요약

| 화면 | 경로 | 역할 |
| --- | --- | --- |
| 로그인 | `/login` | 상담사/관리자 계정 선택 |
| AI 상담 홈 | `/main` | 상담사: 콜 인입·바로가기 / 관리자: KPI 대시보드·모니터링 |
| 실시간 고객 상담 | `/insight-chat?agent=assistant&feature=counseling` | STT·RAG 추천·키워드·관리자 코칭 |
| 상담 이력 조회 | `/post-consultation` | 상담 테이블 탐색 + 상세 카드 |
| 접촉이력 등록 | `/post-consultation?task=contact` | 유형 분류·요약 초안·접촉이력 등록 |
| SMS 안내 작성 | `/post-consultation?task=sms` | 문자 초안 생성·편집·발송 |
| 상담 검수 결과 | `/post-consultation?task=audit-result` | 오안내·안내 누락 AI 검수 + 휴먼 검수 |

### 테이블 셀 바로가기(공통)
상담 이력 조회 테이블에서 아래 상태 셀에 **호버하면 화살표(→)**가 나타나고, 클릭하면 해당 처리/검수 화면으로 바로 이동합니다.

| 셀 | 상태 | 이동 |
| --- | --- | --- |
| 접촉이력 | 미등록 | 접촉이력 등록 |
| SMS | 발송요청 | SMS 안내 작성 |
| AI 검수 | 오안내·누락 감지 | 상담 검수 결과 |
