# DEMO_SPEC — 시연 흐름 & 목업 데이터 정본 (Single Source of Truth)

> **이 문서가 실제 앱 화면의 데모 목업 데이터·시나리오의 "정답"입니다.** (PPT용 아님 — 실제 화면 데이터 통일·관리용)
> 코드 주석이 아니라 **이 문서 기준**으로만 관리합니다.
>
> **동반 문서**: 데이터 스키마·필드·허용값·불변식(메타데이터)은 [DUMMY_DATA_SPEC.md](./DUMMY_DATA_SPEC.md).
> - **DEMO_SPEC** = 시연 흐름 + **화면 전환별로 어떤 데이터가 어떻게 보여야 하는지**(§2·§3 워크스루) + LOCKED 값.
> - **DUMMY_DATA_SPEC** = 데이터 사전(엔티티·필드·타입·enum·화면 간 공유 엔티티).

## 0. 관리 원칙 (반드시 지킬 것)

1. **변경은 이 문서 먼저.** 목업 데이터·시나리오 순서·건수·ID·케이스를 바꾸려면 **여기부터 고치고** 코드를 맞춥니다. 코드 주석 레벨의 임의 변경 금지.
2. **손대지 말 것(LOCKED)** 표시가 있는 값은 시연 대본에 물린 값이므로 지시 없이 바꾸지 않습니다.
3. **날짜는 상대값**으로만 관리(§1.1). 고정 날짜 신규 도입 금지(ID 제외).
4. 값을 바꾸면 **§9 변경 로그**에 한 줄 남기고, 스키마가 바뀌면 **DUMMY_DATA_SPEC도 동기화**합니다.
5. "확인 필요" 항목은 사용자 확정 전까지 임의로 정하지 않습니다.
6. **화면 간 일관성 우선**: 같은 엔티티(§0 공유 엔티티, DUMMY_DATA_SPEC §0)는 모든 화면에서 동일하게 보여야 함. 한 화면만 고치지 말 것.
7. **워크스루 형식**: 시나리오는 `단계 → 트리거(진입·클릭) → 화면/URL → 이 시점에 보여야 하는 데이터` 표로 관리(§2·§3의 "화면 전환 워크스루").

## 1. 전역 규칙

### 1.1 날짜 Remap (동적 날짜) — LOCKED
- 목업의 고정 날짜는 **오늘 기준 상대 간격을 보존**해 매핑. 고정 날짜 그대로 표시 금지.
- **상담 이력계** (`components/PostConsultationView.tsx`)
  - 기준일 `ORIG_TODAY = 2026-05-14`, `remapDate(orig)`가 `(orig − ORIG_TODAY)` 일수를 오늘에 더해 변환.
  - 예: `2026-05-14 → 오늘`, `2026-05-13 → 어제`, `2026-05-08 → 6일 전`.
  - 기간 컷오프: 오늘 / 최근 7일(`remapDate("2026-05-07")`) / 최근 30일(`remapDate("2026-04-14")`).
- **VoC/민원 탐지계** (`components/ComplaintDetectionView.tsx`)
  - 기준일 `CD_BASIS = 2026-06-13`, `remapCDDate()`가 월·일만 오늘 기준 상대 이동(시간은 유지).
- **ID에 박힌 날짜는 remap 대상 아님** (예: `CL-20260514-018`은 문자열 그대로 유지).
- ⚠️ **확인 필요**: 대외민원(FSS) `ymdDeadline`(예: 2026-07-07)이 현재 remap 대상인지 — 실제 오늘과 근접하면 D-Day 배지가 흔들릴 수 있음. 규칙 확정 필요.

## 7. 데모 세션 E2E 시나리오 (상담사) — LOCKED

### 세션 정의 — 구현됨
- **세션 = 브라우저 새로고침 기준.** 새로고침 시 데모 상태는 **baseline(변경 전)으로 복귀**(진행 중 후속처리 전체 초기화). 새로고침 없는 SPA 화면 이동에서는 처리 상태 **유지**.
- 구현: `lib/demo-session.ts`의 `resetDemoFollowupOnce`(모듈 플래그 = 새로고침당 1회) → **앱 진입점 `layout-content`에서 호출**(어떤 후속처리보다 먼저 실행 → DS-2/DS-3 안 깨짐). 새로고침 시 `genon:followup` 전체를 `{}`로 초기화.
- 저장소: `genon:followup`(localStorage) = 후속처리 상태(도트/상세 표시 소스). `genon:pc:filter`·`genon:pc:seen`(sessionStorage) = 필터·세션 플래그.

### DS-1 · 최초 진입 흐름
- 상담사 로그인 → 세션 첫 진입: **실시간 고객 상담(김민준) 완료 → 상담 이력 조회 진입**.
- 상담 이력 조회 **필터 기본 = 오늘**. *(구현됨: period 기본값 `today`, `genon:pc:filter` 세션값 우선)*

### DS-2 · 실시간 상담 중 후속처리 → 이력 반영
- 실시간 상담 화면에서 **접촉이력 등록 / SMS 발송**을 처리하면, 상담 이력 조회 **테이블의 후속업무 도트** + **상세 카드**에 완료 상태가 반영. *(구현됨: `genon:followup` 저장 + `genon:followup-changed` 이벤트로 테이블·상세 즉시 갱신)*

### DS-3 · 후속업무 화면에서 처리 → 이력 반영
- 접촉이력 등록 / SMS 발송 **화면에서 직접 처리**해도 동일하게 이력 테이블/상세에 반영. *(구현됨: 동일 `genon:followup` 경로)*

> ✅ **세션 리셋 보정 완료(2026-07-10)**: 리셋을 앱 진입점(`layout-content`)으로 옮겨 "새로고침=baseline 복귀 / 세션 중 이동=유지"를 구현. (기존 sessionStorage `genon:demoReset` 가드 방식 제거)
