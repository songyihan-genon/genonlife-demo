# 잠재민원 탐지 통합 — 화면 분석 & 설계 메모

> 목표: 관리자가 **콜·이메일·대외기관 문의를 종합 모니터링 → 민원 발전 가능 건 유형 분류 → 담당 부서 이관**.

## 4개 원본 화면 비교 요약

| 항목 | ① 잠재민원 탐지(신규) | ② 고객민원탐지(DetectScreen) | ③ VoC 분석·조회(Admin/Unified) | ④ 콜상담 VoC(AI VoC) |
|---|---|---|---|---|
| 채널 | 콜·챗봇·이메일 | 콜 STT 단일 | 콜+이메일+챗봇+대외기관(INFLOW) | 콜·이메일·대외기관 |
| 감지근거 | riskScore 임계 | **불편도 0–100·STT cues·신호·제도개선영역** | risk/score·cause | score·triggers·senti·문장분석 |
| 부서 이관 | 부서 select+이관 | 없음(매핑만) | **select·일괄·알림·상태흐름·SLA·칸반** | 알림 토글(하드코딩) |
| 대시보드 | KPI+도넛3 | 추이·상품·유형위험·주의유형·**AI평가기준** | 문의구성/유형/위험/처리현황·**3뷰** | analyze 카드 |
| 상세 | 감지→분류→이관(직관) | **STT원문·불편도·등록게이트** | 위험·AI분류·요약·부서·상태·**대외기관 회신** | 위험·문장분석·후속(상담사형) |
| 회신/리포트 | 없음 | 없음 | reply링크·대외기관 textarea | **템플릿5+과거사례·이슈모니터링·리포트** |

## 통합 설계 (3축)

### 1) 대시보드 (중복 제거·선별)
- 유지: KPI 리드아웃 / 채널별 인입·감지(bars) / VoC 유형·위험비중(VocRiskBars) / 위험등급 도넛 / 민원탐지 추이(주·월)
- 보조(1): 실시간 이슈 + 급증 키워드 TOP (carousel 1카드)
- 제거: 키워드 클라우드(급증 TOP과 중복), 문의구성 도넛(채널 bars로 흡수), 상품 도넛(부차), 부서 처리현황→이관 후 보조 탭으로 분리

### 2) 큐 UX
- 기본 = **우선순위 인박스(테이블)**, 칸반은 토글 보조(컬럼=부서/상태)
- 컬럼: □ · 인입(채널·ID·시각) · 고객 · VoC유형(대/소) · 잠재위험(score+bar) · 긴급·SLA · AI추천부서 · 상태 · (행호버)이관→
- 정렬 기본 SLA 임박순, 일괄 부서 배정 지원
- 레이아웃: 상단 대시보드(접기) → 필터바 → 테이블 → 행클릭 우측 드로어

### 3) 상세 드로어 (① 카드 기반 + 흡수)
- 헤더: 채널·상담ID·고객·시각·상태
- 1 감지 근거: 위험게이지·긴급/SLA·트리거칩·**STT 원문 cues**·불편도/감정·제도개선 신호
- 2 AI 1차 분류: 대/소 유형(수정)·고객경험·핵심요약·키워드·**고객 요구사항(NEED)**
- 3 부서 이관: AI추천 부서(수정)·[이관·담당자 알림]·상태흐름
- 4 채널별 후속: 대외기관→금감원 회신 초안 / 콜→VOC 등록 게이트

### 제거할 중복
- AdminAnalyze(VocService) ↔ UnifiedVocAnalysis ↔ AdminAnalyze(CDV) → **단일 인입 큐로 통합**
