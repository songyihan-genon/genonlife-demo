# 고객상담 어시스턴트 — 시나리오 스크립트 명세

> 대상 컴포넌트: [`components/CounselingAssistantView.tsx`](../components/CounselingAssistantView.tsx)
> 진입 경로: 좌측 네비 `민원 상담 중 활용 > 고객상담 어시스턴트`
> (`/insight-chat?agent=assistant&feature=counseling`)

이 화면은 **유저 데모용 자동재생 화면**입니다. 진입하면 STT 대화가 처음부터 한 줄씩 떠오르고, 각 발화에 맞춰 화면 곳곳의 업무지원이 **순서대로 반응**합니다. 마지막 발화 후 잠시 뒤 전체가 초기화되어 **처음부터 무한 반복**됩니다. (재생/정지 버튼 없음 — 실제 라이브 화면처럼)

## 1. 화면 레이아웃 (상단 컨텍스트 바 + 4분할)

```
┌──────────────────────────────────────────────────────────────────────────┐
│ 컨텍스트 바 : 김민준 고객 / 상담ID / 이력 플래그(실효·안내장·접수중)         │
├──────────┬──────────┬───────────────────────┬──────────────────────────────┤
│ 1) STT   │ 2) 가이드 │ 3) 상담지식 어시스턴트  │ 4) 우측 패널 (상하 분할)        │
│ (≈1.15)  │ (≈1.15)  │   RAG 챗봇 (≈2.5)      │   (≈1.75)                     │
│          │          │                       │  ┌─ 업무정보 ──────────────┐  │
│ 말풍선    │ FCR 확인  │  질문→탐색단계→답변     │  │ 계약선택/처리가능/유의사항 │  │
│ +키워드   │ 상담스크립트│ (약관요약·업무기준)    │  └──────────────────────────┘  │
│  하이라이트│ 키워드FAQ │  FAQ + 입력창          │  ┌─ Next Step·작업 추천 ───┐  │
│ +다중선택 │          │                       │  │ 업무화면 / SMS·접촉이력   │  │
└──────────┴──────────┴───────────────────────┴──────────────────────────────┘
```

- **반응형**: 가이드는 `lg`, 우측 패널은 `xl` 이상에서 노출. STT·RAG는 항상 노출.
- **컨텍스트 바**(`ContextBar`): 콜 인입 시 고객정보 + 최근 이력 **플래그**가 떠오름.
- **① STT**: 화자별 말풍선(한 줄씩 등장). 고객 발화에서 추출한 **키워드를 버블 본문에 하이라이트**(노란 마크). 고객 버블 클릭 시 다중선택 → 복합 질의.
- **② 상담 가이드**: `FCR 필수 안내 확인`(자동체크) · `추천 상담 스크립트`(대화 유형별) · `대화 키워드·FAQ`(추출 키워드 + 추천 질의).
- **③ 상담지식 어시스턴트(RAG)**: **약관 요약·업무 기준 등 지식성 질의에만** 사용. `질문 → 탐색 4단계 → 답변(요약·근거·출처·업무기준)`.
- **④ 우측 패널**: 위 `업무정보`(계약 연동 데이터) + 아래 `Next Step·작업 추천`.

### ⚠️ RAG vs 업무정보 — 역할 분리 (중요)
- **RAG(상담지식)** = 약관/지침 등 **지식 검색**. "필요 서류 *기준*", "자동이체 변경 *기준*", "해지환급금 안내 *기준*"처럼 일반 기준에 적합.
- **업무정보** = 계약·처리계 **연동 데이터**. "고객이 무엇을 제출했는지(제출/미제출)", "이 계약에서 처리 가능한지", "예상 환급금/한도/변액" 등 **연동 없이는 RAG가 답할 수 없는 정보**를 담당.
- 예) 실손 추가서류: RAG는 *약관상 필요 서류 목록*을, 업무정보는 *고객의 실제 제출 현황(진료비계산서 제출 / 세부내역서 미제출)* 을 보여줌.

## 2. 시나리오 엔진 동작 원리

핵심 데이터는 `SCENARIO` 배열([CounselingAssistantView.tsx](../components/CounselingAssistantView.tsx))이며, 각 원소가 **대화 1턴 + 연동 반응**을 정의합니다.

```ts
interface ScenarioStep {
  bubble: { speaker: "customer" | "agent"; time: string; text: string }
  highlightInBubble?: string[]  // 버블 본문에서 노란색으로 강조할 실제 표현
  keyword?: string              // 추출 키워드 (가이드 키워드칩 등록 + 자동 선택)
  query?: string                // RAG 지식 질의 (지식성 질의만)
  effects?: {
    checks?: string[]    // FCR 자동 체크 항목 id
    flags?: string[]     // 컨텍스트 바 이력 플래그 id
    topic?: "claim" | "autodebit" | "surrender"  // 업무정보/추천 컨텍스트
    contract?: string    // 업무정보에서 선택할 계약번호
    script?: string       // 가이드 추천 스크립트 key
    highlight?: "context" | "guide" | "rag" | "work" | "next"  // 강조 영역
  }
  delayAfter: number     // 다음 스텝까지 대기(ms)
}
```

드라이버(`useEffect([step])`)가 `step`을 0→1씩 진행시키며 효과를 타이머로 실행합니다. 마지막 스텝 후 `LOOP_PAUSE`(5.5초) 뒤 전체 상태를 초기화하고 `step=0`으로 **무한 루프**. 모든 `setTimeout`은 `timers` ref에 모아 두고 스텝 전환·언마운트 시 정리(중첩 방지).

### 스텝 내부 타이밍 (기준 = 스텝 시작)

| 시점 | 동작 |
|------|------|
| `0ms` | 말풍선 노출(키워드 버블 하이라이트 포함), 키워드 칩 등록, 플래그/계약/토픽/스크립트 반영 |
| `600ms` | 키워드 **자동 선택**(파란 펄스) |
| `600 + i*400ms` | FCR 체크 i번째 자동 체크(에메랄드 펄스, 1.3초 후 해제) |
| `ragOffset` (질의 有 `1300ms` / 無 `500ms`) | 영역 하이라이트 시작 |
| `1300ms` | 키워드 선택 해제 → **RAG 질문 전송** |
| `2500ms` | **RAG 답변 출력** |
| `ragOffset + 2200ms` | 하이라이트 해제 |
| `delayAfter` | 다음 스텝 진행 |

## 3. 단계별 상세 스크립트 (현재 6스텝)

> 🗣️ STT 발화 / 🔑 키워드(버블 강조) / 🤖 RAG(지식) / 🧩 업무정보 / 🎯 가이드·플래그 / ➡️ 추천 / ⏱️ 소요

### STEP 0 — 상담 인입 · 본인확인  ⏱️ 4.0초
- 🗣️ **상담사**(10:21): "안녕하세요, 신한라이프 상담사 이서연입니다. 통화는 녹취되며, 본인 확인 후 도와드리겠습니다."
- 🎯 컨텍스트 바 강조 + 이력 **플래그 3종 노출**(`실효 이력 3건`/`갱신 안내장 수령`/`보험금 접수 중`), FCR 자동 체크(`본인 확인`→`녹취 고지`→`개인정보 동의`), 가이드 스크립트 = **도입·본인확인**
- **주목**: 콜 인입 즉시 Tele-Pro 고객정보·이력 플래그·필수 안내가 자동 세팅.

### STEP 1 — 고객 1차 질의: 보험금 청구 추가서류  ⏱️ 5.6초
- 🗣️ **고객**(10:21): "보험금 청구를 했는데 **〔추가 서류〕** 가 필요하다고 문자가 왔어요…" (버블 내 `추가 서류` 하이라이트)
- 🔑 키워드 `추가서류` 추출 → 자동 선택
- 🤖 RAG 질의 "실손 보험금 청구 시 필요 서류 기준" → **약관상 필요 서류 기준** 답변
  - 요약: 진료비계산서·세부내역서 기본, 고액·특정 질환 시 진단서 / 출처: 실손 약관 제12조, 청구 업무매뉴얼 v3.2
  - 업무 기준: "서류 기준은 약관 기준이며, 실제 제출/미제출 현황은 업무정보 확인"
- 🧩 **업무정보**(계약 `SL-2048-5521` 실손 선택) → **청구 서류 제출 현황(처리계 연동)**: 진료비계산서 `제출 완료`, 세부내역서 `미제출`, 진단서 `미제출`
- 🎯 FCR `추가서류·처리절차 안내` 자동 체크, 가이드 스크립트 = 청구 서류 안내
- **주목**: RAG는 *기준*을, 업무정보는 *제출 현황*을 분담 → 역할 분리 시연.

### STEP 2 — 상담사 안내(세부내역서)  ⏱️ 4.0초
- 🗣️ **상담사**(10:22): "확인해 보니 진료비 세부내역서가 미제출 상태입니다. 병원 원무과 발급 후 앱으로 제출해 주시면 됩니다."
- ➡️ **Next Step 강조**: `보험금 청구 진행조회(CLM-201)`, `추가서류 접수 등록(CLM-205)` + 후속 작업(SMS 안내 발송·접촉이력 등록)
- **주목**: 답변 직후 다음 업무 화면/후속 작업 추천.

### STEP 3 — 고객 2차(복합) 질의: 자동이체 계좌 변경  ⏱️ 5.6초
- 🗣️ **고객**(10:22): "그리고 보험료 **〔자동이체 계좌〕** 도 바꾸고 싶은데 가능할까요?" (하이라이트)
- 🔑 키워드 `자동이체 계좌변경` 추출 → 자동 선택
- 🤖 RAG 질의 "보험료 자동이체 계좌 변경 기준" → **변경 기준** 답변(본인 명의 원칙, 5영업일 전 신청 당월 적용, 카드납 전환 가능 / 출처: 수금·자동이체 업무지침 제8조)
- 🧩 **업무정보**(계약 `SL-1180-3372` 종신 선택) → 처리 가능 여부(`자동이체 계좌 변경: 가능`, `카드납 변경: 가능(신용·체크)`, `정상 입금: 가능`) + 주요 정보(월보험료/해지환급금 예상/변액 수익률) + 유의사항(정상 납입 중 등)
- **주목**: 복합 질의를 1차 청구 건과 분리 처리, 계약 기반 처리 가능 여부 판단.

### STEP 4 — 상담사 처리 가능 확인  ⏱️ 4.0초
- 🗣️ **상담사**(10:23): "네, 본인 명의 계좌로 변경 가능합니다. 처리 가능 여부와 유의사항 확인해 드리겠습니다."
- 🧩 업무정보 강조 → **처리 시 유의사항**(정상 납입 중, 실효 후 부활 이력, 해지방어 등) 사전 조회.

### STEP 5 — 고객 3차 질의: 해지환급금  ⏱️ 5.6초
- 🗣️ **고객**(10:23): "예전에 가입한 종신보험을 **〔해지〕** 하면 **〔환급금〕** 이 얼마나 나오는지도 궁금해요." (하이라이트)
- 🔑 키워드 `해지환급금` 추출 → 자동 선택
- 🤖 RAG 질의 "종신보험 해지환급금 안내 기준" → **안내 기준** 답변(산출식, 보장 종료·재가입 제한 고지, 해지방어 우선 / 출처: 종신 약관 제30조, 해지방어 가이드)
- 🧩 업무정보(종신) → 해지환급금 예상액/변액 정보, 해지 `가능(해지방어 안내 필수)`
- ➡️ **Next Step 강조**: `해지환급금 조회(POL-330)`, `해지방어 캠페인(CMP-052)`
- **주목**: 해지 키워드 감지 시 해지방어 가이드와 조회 화면을 함께 추천.

### LOOP — 재시작  ⏱️ 5.5초 후
모든 상태(말풍선·채팅·체크·플래그·하이라이트·토픽·계약)가 초기화되고 STEP 0부터 다시 재생.

## 4. RAG 답변 매칭 규칙

`generateAnswer(query)`는 질의 키워드로 답변을 분기합니다(데모용 규칙 매칭, 지식성 답변만 반환).

| 우선순위 | 매칭 정규식 | 반환(지식) |
|---------|------------|-----------|
| 1 | `자동이체 \| 계좌 \| 납입 \| 카드` | 자동이체 변경 **기준** |
| 2 | `해지 \| 환급 \| 해약` | 해지환급금 안내 **기준** |
| 3 (기본) | 그 외 | 실손 청구 **필요 서류 기준** |

> 고객의 실제 제출/처리 현황 등 **연동 데이터는 RAG가 아닌 업무정보 패널**(`contractDetail`)에서 제공.

## 5. 데이터 정의 위치 (수정 시 참고)

모두 [`components/CounselingAssistantView.tsx`](../components/CounselingAssistantView.tsx) 한 파일.

| 항목 | 식별자 | 설명 |
|------|--------|------|
| 대화 시나리오 | `SCENARIO` | 스텝(발화·버블강조·키워드·질의·효과·딜레이) |
| 루프 대기 | `LOOP_PAUSE` | 마지막 후 재시작 대기(ms) |
| RAG 지식 답변 | `generateAnswer()` | 질의→지식 답변 매칭/본문 |
| FAQ 칩 | `faqPresets` | 가이드/입력창 추천 질의 |
| FCR 체크 | `checklistSeed` | 필수 안내 항목 |
| 추천 스크립트 | `scripts` | 대화 유형별 상담 스크립트(`intro`/`claim`/`autodebit`/`surrender`) |
| 이력 플래그 | `flagMeta` | 플래그 라벨/색상 |
| 보유 계약 | `contracts` | 계약번호/상품명/상태 |
| 업무정보 상세 | `contractDetail` | 계약별 figures(금액/한도/변액)·process(처리 가능)·cautions(유의)·claimDocs(청구 서류 현황) |
| Next step 화면 | `nextStepsByTopic` | 토픽별 다음 업무 화면 |

### 새 대화 스텝 추가 예시
```ts
{
  bubble: { speaker: "customer", time: "10:24", text: "보험금 지급은 언제 되나요?" },
  highlightInBubble: ["보험금 지급"],
  keyword: "지급 시기",
  query: "보험금 지급 예상 시기 기준",   // 지식성이면 RAG, 연동 데이터면 업무정보로
  effects: { topic: "claim", contract: "SL-2048-5521", highlight: "work" },
  delayAfter: 5600,                       // RAG 스텝 ≥5600 권장(질문 1.3s + 답변 1.2s + 여유)
}
```
- 단순 안내(상담사) 스텝은 `delayAfter ≈ 4000`.
- `bubble`을 추가하면 `SCENARIO.length` 증가 → 자동으로 마지막+루프 처리.

## 6. 수동 인터랙션 (자동재생과 공존)
